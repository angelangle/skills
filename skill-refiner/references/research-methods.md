# 科研方法论

## 1. 研究流程

### 1.1 标准研究循环

```
观察现象（Observation）
    ↓
提出问题（Question）
    ↓
文献调研（Literature Review）
    ↓
假设形成（Hypothesis）
    ↓
实验设计（Experiment Design）
    ↓
数据收集（Data Collection）
    ↓
分析验证（Analysis）
    ↓
结论提炼（Conclusion）
    ↓
知识整合（Integration）
    ↓
[回到观察，循环往复]
```

### 1.2 快速研究模式

对于Skill增强的轻量级研究：

```
识别知识缺口
    ↓
快速检索（WebSearch）
    ↓
交叉验证（2-3个来源）
    ↓
提炼要点
    ↓
结构化输出
    ↓
整合到Skill
```

## 2. 文献调研方法

### 2.1 检索策略

**学术资源**：
```
Google Scholar
    ├─ "关键词" + "survey"（综述）
    ├─ "关键词" + "review"（评论）
    └─ 作者筛选（领域大牛）

arXiv
    ├─ 最新预印本
    ├─ 按时间排序
    └─ 关注高引论文

ACM Digital Library / IEEE Xplore
    ├─ 顶级会议（SOSP/OSDI/PLDI）
    └─ 期刊论文
```

**工业资源**：
```
官方文档（优先Primary Source）
    └─ 一手信息，最权威

大厂技术博客
    ├─ Google Engineering Blog
    ├─ Netflix Tech Blog
    ├─ Meta Engineering
    └─ 阿里云/腾讯云/字节跳动技术博客

开源项目
    ├─ GitHub Awesome列表
    ├─ 核心项目文档
    └─ Issue和PR讨论
```

**社区资源**：
```
Stack Overflow
    └─ 高票回答，实战经验

Reddit / Hacker News
    └─ 趋势讨论，前沿动态

专业论坛
    └─ 深度技术讨论
```

### 2.2 文献评估框架

| 维度 | 评估标准 | 权重 |
|-----|---------|------|
| **权威性** | 来源可信度、作者资质 | 30% |
| **时效性** | 发布时间、更新频率 | 25% |
| **相关性** | 与目标问题的匹配度 | 25% |
| **实用性** | 可操作程度 | 15% |
| **共识度** | 业界普遍认可程度 | 5% |

## 3. 知识提炼技术

### 3.1 信息抽取

**关键要素识别**：
```
概念定义：核心术语的准确定义
方法流程：步骤化的操作指南
对比分析：不同方案的优缺点
最佳实践：经过验证的做法
常见陷阱：需要避免的错误
工具资源：可用的辅助工具
```

### 3.2 结构化方法

**思维导图法**：
```
中心主题
    ├─ 分支1：概念
    │   ├─ 子概念1
    │   └─ 子概念2
    ├─ 分支2：方法
    │   ├─ 方法1
    │   └─ 方法2
    └─ 分支3：应用
        ├─ 场景1
        └─ 场景2
```

**表格对比法**：
```
| 方法 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| A   | ...  | ...  | ...     |
| B   | ...  | ...  | ...     |
```

**流程图法**：
```
决策流程、执行步骤、依赖关系
```

### 3.3 批判性思维

**验证清单**：
- [ ] 信息来源是否可靠？
- [ ] 是否有多个来源交叉验证？
- [ ] 是否存在利益相关导致的偏见？
- [ ] 是否考虑了反例和边界情况？
- [ ] 结论是否有过分泛化？
- [ ] 数据样本是否足够？

## 4. 知识整合策略

### 4.1 层次化组织

```
Level 1: 核心概念（必须掌握）
Level 2: 扩展知识（深入了解）
Level 3: 参考资料（按需查阅）
Level 4: 进阶阅读（专家级别）
```

### 4.2 渐进式披露

在Skill中实现：
```markdown
## 快速开始（Quick Start）
[最核心的信息，80%场景适用]

## 详细指南（Detailed Guide）
[完整信息，覆盖所有场景]

## 深入原理（Deep Dive）
[理论背景，为什么这样做]

## 参考资料（References）
[外部链接，延伸阅读]
```

### 4.3 关联建立

**横向关联**：
- 相似概念对比
- 相关工具链接
- 互补方法引用

**纵向关联**：
- 基础概念 → 高级应用
- 历史演进 → 当前状态 → 未来趋势

## 5. 科研写作规范

### 5.1 技术文档结构

```markdown
# 标题

## 摘要（Abstract）
- 一句话总结
- 适用读者
- 核心收获

## 1. 引言（Introduction）
- 背景
- 问题定义
- 目标

## 2. 核心内容（Body）
- 概念定义
- 方法论述
- 案例分析
- 对比讨论

## 3. 实践应用（Application）
- 使用场景
- 操作步骤
- 注意事项

## 4. 总结（Conclusion）
- 要点回顾
- 行动建议
- 延伸阅读
```

### 5.2 写作风格

**清晰性原则**：
- 一句话一个意思
- 段落有主题句
- 使用列表和表格

**准确性原则**：
- 术语定义明确
- 区分事实和观点
- 引用来源标注

**可读性原则**：
- 适当使用图表
- 代码示例完整
- 难度循序渐进

## 6. 自动化研究工具

### 6.1 信息检索自动化

```python
# 研究助手伪代码
class ResearchAssistant:
    def research(self, topic):
        # 1. 生成查询
        queries = self.generate_queries(topic)

        # 2. 并行检索
        results = []
        for query in queries:
            results.extend(self.search(query))

        # 3. 去重排序
        unique_results = self.deduplicate(results)
        ranked = self.rank_by_relevance(unique_results)

        # 4. 深度获取
        detailed = []
        for result in ranked[:10]:
            detailed.append(self.fetch_full_text(result))

        # 5. 提炼总结
        summary = self.summarize(detailed)

        return summary
```

## 7. 研究伦理

### 7.1 引用规范

- **直接引用**：加引号并标注来源
- **改写引用**：用自己的话重述，仍需标注
- **观点引用**：明确区分他人观点和自己观点

### 7.2 准确性责任

- 不确定的信息要标注
- 过时的信息要更新
- 错误的信息要修正

### 7.3 知识产权

- 尊重原作者版权
- 合理使用范围
- 优先使用开放资源
